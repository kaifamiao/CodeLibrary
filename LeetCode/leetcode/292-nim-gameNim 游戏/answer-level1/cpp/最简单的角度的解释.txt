其实这样想就很简单，1,2,3,一定是叫你拿来凑4的。n是4的的倍数的情况下无论怎么拿后手的人只要和先手的人上次拿的加起来凑够4，则n/4轮后刚好拿完。
n=4k
      a     b
 1回合 1    3
 2回合 3    1
 3回合 2    2
 4回合 3    1
       ....
 k回合 x   4-x  
**所以n是4的倍数先手一定输。**
**再说为什么n不是4的倍数先手一定赢**
 因为先手的可以拿1到3个，n不是4的倍数则0<n%4<4,第一步只要拿走n%4后剩下的石子一定是4的倍数，所以就变成了4的倍数下对方先手拿石子的问题我们跟着凑4就赢了


class Solution {
public:
    bool canWinNim(int n) {
        return (!((n-1)%4)||!((n-2)%4)||!((n-3)%4));
    }
};

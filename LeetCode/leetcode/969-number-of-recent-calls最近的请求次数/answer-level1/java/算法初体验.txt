# 题意
这道题一开始不是很明白题意，看了一下评论。很多人说是阅读理解问题。后来才知道，其实就是计算一个时限来ping的次数。

# 解题
第一个想法是用list保存所有的ping，然后再计算差距小于3000的。当然这个是直观的思路，也是不成功的原因。因为太没有注重效率，题意中其实有一个隐含，就是当差距大于3000后则不需要关注。
另外，在用list解题时也遇到一个问题，就是怎样保存多次ping的问题。list的结构其实不适合在有数据后再操作，因为一边增加元素，一边减少元素的操作不可取，很混乱。所以后来第二个版本又通过ietrator去处理。第三个版本则想到了用数据结构stack解决，因为考虑当stack弹出的元素超过3000后不需要再处理。

# 思路
最有感觉的是思维的迭代，由list 到 ietrator，到stack。感觉到算法和数据结构果然是分不开的。

### 解题思路
实现的第一想法是递归，但是当递归到达四五十时，运算量急剧增大。
基本思路：通过解析一个例子发现递归过程中有很多重复计算的节点，因此只要把这些节点存储起来，即可减少这些运算。
小技巧：
1、一开始想着n有多大就存多大，但是进一步发现，其实只存两个f(n-1)和f(n-2)，然后每次都更新这俩就够用了。
2、题目中答案取模是控制了int的大小范围，因此每次更新计算都别忘了取模。

### 代码

```cpp
class Solution {
public:
    int fib(int n) {
        int a[] = {0,1};
        int tmp ;
        for(int i = 0;i<n;i++)
        {
            tmp = (a[0]+a[1])%(1000000007);
            a[0] = a[1];
            a[1] = tmp;
        }
        return a[0];
    }
};
```

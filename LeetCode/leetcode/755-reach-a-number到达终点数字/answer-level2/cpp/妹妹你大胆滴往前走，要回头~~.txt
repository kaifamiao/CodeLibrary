首先，由于是从原点出发，故而对于目标处于正半轴还是负半轴的处理思路是一样，为了方便处理，我们都将目标点改为正值。

之后，想要达到目标点，我们必然要朝着目标方向进发，也就是妹妹你大胆滴往前走。但是问题来了，当快要到达目标点时，这时妹妹的一步就可能很大了。俗话说，步子大了容易闪着腰，很可能妹妹这后面的一步迈出去就跨过目标点了。这可咋整？妹妹柳眉一皱，发现事情并不简单。

俗话又说的好，回眸一笑百媚生。妹妹心中暗生一计，我这回眸迷不住那呆子，可能往后退啊，这一来一回不就相当于步子迈的小了么，对就是赵老师说的那样，三步一窜呀么两呀两回头，五步一下腰，六步一招手，然后你再趟呀么趟着走。妹妹回头撤三米，再往前走四米，不就把原来能走七米的距离压缩到往前一米了么。

既然想到了压缩步伐的办法，妹妹这又开始大胆的往前走了起来。当妹妹最后一个飞跃走完最后一步时，已然超过了目标点。这时问题又来了，超是超过了，可超了奇数米，要是个偶数，不刚好可以在之前的某两步中把这多出来的压缩了么，这可又咋整？怎么才能超出偶数米呢，伤脑筋啊伤脑筋。。。。桥豆麻袋！老娘这步长不就奇偶交替么！对啊，我这继续往前走，凑出个偶数便是了嘛。
```
class Solution {
public:
    int reachNumber(int target) {
        int n=0;
        target=abs(target);
        while((1+n)*n/2<target && ++n);        
        if(((1+n)*n/2-target)%2==0)
            return n;
        else            
            return n%2==1?n+2:n+1;            
    }
};
```

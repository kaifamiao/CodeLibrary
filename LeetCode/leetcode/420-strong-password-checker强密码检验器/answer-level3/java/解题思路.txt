所有操作一共有三种：插入、删除、修改

- 少于6个字符的情况，插入优先级最高，然后修改，不要删除
- 6~20个字符的情况，修改优先级最高，然后插入
- 最麻烦的是最后这种，大于20个字符的情况，删除优先级最高，然后修改，不要插入，麻烦的点在于修改会影响需要删除的字符，删除同样会影响需要修改的字符。

针对最后一种情况，我的思路是
1. 通过插入先补充字符类型的缺失，因为这个是最必要的。
2. 然后修改字符列表，调整连续字符的个数。
3. 删除连续字符，从连续个数为3n的开始删除，因为效率最高，然后删除3n+1的，然后删除3n+2的，最后这个可以三个三个的删。
4. 删到长度不再超标，如果还有连续字符，再用修改操作。
